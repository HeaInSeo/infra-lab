# infra-lab 사용법

이 문서는 `infra-lab`으로 Kubernetes 클러스터를 생성하고 운영하는 절차를 설명한다.

---

## 사전 요구사항 확인

```bash
cd /path/to/infra-lab
ilab doctor
```

다음 항목이 모두 `✓` 이어야 한다.

| 항목 | 용도 |
|------|------|
| `git` | 루트 감지, 메타데이터 |
| `tofu` | VM 생성/삭제 |
| `kubectl` | 클러스터 접근 |
| `virsh` | libvirt 백엔드 VM 목록 |
| `qemu-img` | libvirt 디스크 이미지 |
| `jq` | tfstate에서 IP 자동 추출 |

libvirt 그룹 멤버십도 확인한다.

```bash
id | grep libvirt
```

출력에 `libvirt`가 없으면 추가한다.

```bash
sudo usermod -aG libvirt $USER
exec sg libvirt newgrp `id -gn`
```

## 환경 프로파일 설정

SSH 키 경로 등 민감한 설정은 `~/.config/infra-lab/` 에 보관한다.
이 위치는 저장소 외부이므로 공개 저장소에 노출되지 않는다.

```bash
mkdir -p ~/.config/infra-lab
```

`~/.config/infra-lab/<이름>.env` 파일을 생성한다.

```bash
# 기본 설정
LAB_HOST_MODE=local
BACKEND=libvirt
CNI=flannel
ADDONS=

# SSH 키 (큰따옴표 필수)
export TF_VAR_ssh_private_key_path=/home/<user>/.ssh/id_ed25519
export TF_VAR_ssh_public_key="ssh-ed25519 AAAA..."

# libvirt 스토리지 풀 위치
export TF_VAR_libvirt_pool_name=lab-pool
export TF_VAR_libvirt_pool_path=/var/lib/libvirt/images
```

공개 키는 `cat ~/.ssh/id_ed25519.pub` 로 확인한다.

## 클러스터 생성

```bash
cd /path/to/infra-lab

HOST_PROFILE=~/.config/infra-lab/<이름>.env \
  bash scripts/k8s-tool.sh up
```

SSH 세션이 끊길 수 있는 경우 nohup 으로 실행한다.

```bash
nohup bash -c '
  HOST_PROFILE=~/.config/infra-lab/<이름>.env \
    bash scripts/k8s-tool.sh up
' > /tmp/infra-lab-up.log 2>&1 &

tail -f /tmp/infra-lab-up.log
```

`up` 은 다음 순서로 실행된다.

1. `tofu init` → `tofu apply` — VM 3개 생성 (master × 1, worker × 2)
2. kubeadm 클러스터 초기화 및 워커 join
3. kubeconfig → `state/<이름>/kubeconfig` 저장
4. 각 VM 에 `/etc/infra-lab/build.json` 기록
5. `metrics-server` base 애드온 설치
6. `state/<이름>/meta` 환경 메타데이터 기록

완료 시 아래 메시지가 출력된다.

```
Apply complete! Resources: 16 added, 0 changed, 0 destroyed.
...
[INFO] environment metadata written: state/.../meta
```

소요 시간: 약 5~10분

## 상태 확인

### 환경 목록

```bash
ilab env list
```

### 클러스터 노드 및 파드

```bash
KUBECONFIG=state/<이름>/kubeconfig ilab k8s status
```

### 리소스 사용량

```bash
KUBECONFIG=state/<이름>/kubeconfig kubectl top nodes
```

### VM 목록

```bash
ilab vm list
```

### VM 빌드 정보

```bash
ilab vm version lab-master-0
ilab vm version lab-worker-0
ilab vm version lab-worker-1
```

---

## VM 접속

```bash
# ilab 으로 접속 (libvirt backend: SSH 사용)
ilab vm ssh lab-master-0

# IP 직접 확인 후 SSH
ilab vm list
ssh ubuntu@<IP>
```

## 클러스터 삭제

```bash
HOST_PROFILE=~/.config/infra-lab/<이름>.env \
  bash scripts/k8s-tool.sh down
```

VM, 디스크 이미지, libvirt 스토리지 풀이 모두 삭제된다.

---

## 재생성 (down → up)

```bash
HOST_PROFILE=~/.config/infra-lab/<이름>.env \
  bash scripts/k8s-tool.sh down

rm -rf state/<이름>

nohup bash -c '
  HOST_PROFILE=~/.config/infra-lab/<이름>.env \
    bash scripts/k8s-tool.sh up
' > /tmp/infra-lab-up.log 2>&1 &

tail -f /tmp/infra-lab-up.log
```

## 문제 해결

### `ilab vm list` 가 libvirt VM 을 표시하지 않는다

virsh 가 `qemu:///session` 에 연결되어 VM 이 보이지 않는 경우이다.

```bash
virsh -c qemu:///system list --all
```

libvirt 그룹 추가 후 세션을 재시작하면 해결된다.

### `kubectl` 이 다른 클러스터에 연결된다

`KUBECONFIG` 환경변수를 명시한다.

```bash
export KUBECONFIG=/path/to/infra-lab/state/<이름>/kubeconfig
kubectl get nodes
```

### SSH 공개 키 관련 오류

env 파일에서 `TF_VAR_ssh_public_key` 값이 큰따옴표로 감싸여 있는지 확인한다.

### libvirt 권한 오류

```bash
sudo usermod -aG libvirt $USER
sg libvirt -c "virsh list --all"
```

## ilab 명령어 요약

```bash
ilab doctor                      # 환경 진단 및 사전 요구사항 확인
ilab env list                    # 관리 환경 목록
ilab env status [env]            # 클러스터/VM 상태
ilab vm list                     # 관리 VM 목록
ilab vm list --all               # 전체 VM 목록 (비관리 포함)
ilab vm version <vm>             # VM 빌드 정보 조회
ilab vm ssh <vm>                 # VM 쉘 접속
ilab k8s status [env]            # kubectl nodes + pods
```
